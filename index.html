RAG-based Agriculture Assistant

Description

  • A domain-specific assistant designed to help farmers and researchers query agricultural PDFs with AI-powered retrieval, making knowledge more accessible and actionable.

Tech Stack

Core Backend

  • FastAPI → lightweight Python web framework for serving your RAG queries.
  • Uvicorn → ASGI server used to run FastAPI inside Docker.
  • Python 3.14.2 runtime version

Machine Learning / RAG

  • Hugging Face embeddings → for converting text chunks into vector representations.
  • ChromaDB → vector database to store and retrieve embeddings.
  • PDF parsing libraries (e.g., PyPDF2, pdfplumber) → to chunk and process agricultural documents.
  • RAG pipeline → query → embedding → vector DB retrieval → response generation.

Bot Integration

  • Telegram Bot API → connects your backend to Telegram users.
  • Python‑telegram‑bot library → handles commands (/start, /ask) and message flow.

Containerization & Deployment

  • Docker → packages backend and bot into reproducible containers.
  • Docker Hub → stores your images (rag-service, telegram-bot).
  • Render → cloud platform that pulls your Docker images and runs them as web services.
  • Environment variables → for secrets (TELEGRAM_TOKEN) and backend URL (RAG_API_URL).

SailPoint–Blockchain Provisioning Log

Description

  • A secure identity management solution that logs provisioning events on the blockchain, ensuring transparency and tamper-proof audit trails for enterprises.

Tech Stack

SailPoint IdentityIQ

  • Core identity governance platform.
  • Provides provisioning workflows and rules (BeanShell/Java).
  • Captures events like create, update, delete.

Middleware Service

  • Built with FastAPI (Python) or Spring Boot (Java).
  • Exposes REST endpoints for SailPoint rules to call.
  • Handles authentication, retries, and payload formatting.

Blockchain Platform

  • Ethereum (public or private) or Hyperledger Fabric (enterprise).
  • Provides ledger immutability and distributed trust.
  • Stores provisioning events as transactions.

Smart Contracts

  • Written in Solidity (Ethereum) or Chaincode (Go/Java) for Hyperledger.
  • Functions like storeProvisioningEvent(identity, operation, timestamp) validate and record logs.
  • Guarantees immutability and auditability.

Audit Dashboard

  • Built with Angular.
  • Displays SailPoint events alongside blockchain transaction IDs.
  • Provides compliance reports and search filters.

Job-Matching Website

Description

  • A smart recruitment platform that uses AI to match candidate skills with job openings, streamlining hiring and improving candidate-job fit.

Tech Stack

Frontend

  • React 19 (with React Router 7 for routing)
  • Tailwind CSS 3
  • Shadcn UI components (Sheet, Select, Input, Sonner toaster, etc.)
  • Cabinet Grotesk (display headings, via Fontshare CDN)
  • IBM Plex Sans + IBM Plex Mono (body + metadata, via Google Fonts)
  • Axios for API calls (with a `jobscope_token` Bearer interceptor from `localStorage`)
  • Sonner for toast notifications
  • CRACO as the build tool (React Scripts wrapper)
  • Yarn package manager (per env requirement)

Backend

  • FastAPI (Python web framework, all routes under `/api`)
  • Uvicorn (ASGI server, managed by supervisor on `0.0.0.0:8001`)
  • Motor (async MongoDB driver)
  • APScheduler 3.11 (daily cron at 06:00 UTC for the auto-refresh)
  • httpx (async HTTP client for the Amazon Jobs public JSON scrape)
  • PyJWT (JWT access token signing, HS256)
  • bcrypt (password hashing)
  • Pydantic v2 (request/response models with EmailStr validation)
  • emergentintegrations (Emergent's LLM wrapper library)

AI / LLM

  • Hugging Face router/ Serverless endpoint and HF_API_KEY

Database

  • MongoDB (running locally inside the container)
  • Collections: `users`, `jobs`, `matches`, `saved_jobs`, `system` (for last-scheduled-refresh state)
  • Indexes: unique `users.email`, unique `jobs.id`, compound `(user_id, job_id)` on matches & saved_jobs

Auth

  • Custom email + password with bcrypt hashing
  • JWT access tokens (7-day expiry), sent via `Authorization: Bearer ` header
  • Token stored in `localStorage` on the frontend

Job Data Sources

  • Amazon Jobs — live public JSON endpoint (`amazon.jobs/en/search.json`)
  • Naukri.com — bot-protected → 8 curated seed listings linking to each company's live careers page

Infrastructure & Tooling

  • Kubernetes container (Emergent's runtime)
  • Supervisor (manages `backend` and `frontend` processes with hot-reload)
  • CORS middleware allowing the frontend origin
  • Posthog (analytics, pre-wired in `index.html`)
  • ESLint + Ruff for JS/Python linting

UPSC Preparation & AI Evaluation Platform

Description

  • A mobile-first app built with Flutter and Firebase that helps UPSC aspirants manage profiles, toggle study themes, reset passwords, and sync progress with a backend API.

Tech Stack

Frontend UI

  • Flutter -> Cross‑platform mobile app framework (widgets, Material UI, state management).

AI agent

  • Hugging Face -> katanemo/Arch-Router-1.5B

Database (primary)

  • Firebase Firestore -> Stores user profile data (users collection with fullName, phone, email)

Database (secondary)

  • MySQL Database -> Stores user's test data and scores.

Authentication

  • Firebase Auth Handles login, logout, password reset

Backend API

  • Node.js/Express (hosted on Render) -> Syncs user profile data with your server DB via REST endpoints (/users/sync-user/:id)

Networking

  • http package -> Makes REST API calls to backend

Local storage

  • SharedPreferences -> Persists theme preference (dark/light mode)

State management

  • Flutter StatefulWidget + setState Manages UI updates after profile edits

How It Works Together

  • Profile data load: _loadUserProfile() fetches data from Firestore.
  • Edit profile: _updateProfile() updates Firestore and syncs with backend DB.
  • Theme toggle: _toggleTheme() saves preference in SharedPreferences.
  • Password reset: _sendPasswordReset() triggers Firebase Auth email reset.
  • Sign out: FirebaseAuth.instance.signOut() clears session and navigates to login.
